# ?? 即时模型加载功能说明

## ?? 功能概述

在工位配置界面中，当用户修改深度学习模型相关配置后，系统会提示用户是否立即加载模型，无需重启程序或重新打开方案。

---

## ? 支持的即时加载场景

### 1. 选择模型路径后
```
用户操作：
PropertyGrid → 深度学习 → 模型文件夹路径 → 选择文件夹

系统响应：
  ↓
弹出确认对话框：
  "检测到模型配置变更：模型文件夹路径
   工位: 工位1
   模型: MyModel
 运行时: GPU 运行时
   
   是否立即加载模型？"
  ↓
[是] → 立即加载模型 → 显示结果
[否] → 稍后手动加载或重启程序时加载
```

### 2. 切换运行时类型后
```
用户操作：
PropertyGrid → 深度学习 → 运行时类型 → 选择 "TensorRT 加速"

系统响应：
  ↓
弹出确认对话框：
  "检测到模型配置变更：运行时类型
   工位: 工位1
   模型: MyModel
   运行时: TensorRT 加速
   
   是否立即加载模型？"
  ↓
[是] → 卸载旧模型 → 加载新运行时模型 → 显示结果
[否] → 保留旧模型，重启后生效
```

### 3. 启用/禁用模型加载
```
用户操作：
PropertyGrid → 深度学习 → 是否加载模型 → ? 勾选

系统响应（启用）：
  ↓
弹出确认对话框：
  "检测到模型配置变更：是否加载模型
   ...
   是否立即加载模型？"
  ↓
[是] → 加载模型
```

```
用户操作：
PropertyGrid → 深度学习 → 是否加载模型 → 取消勾选

系统响应（禁用）：
  ↓
自动卸载模型 → 显示提示：
  "工位[工位1] 模型已卸载"
```

## ?? 完整使用流程

### 场景1: 首次配置模型

```
1. 打开工位配置界面
   ├── 菜单: 工艺 → 工位配置
   └── 或快捷按钮

2. 选择目标工位
   └── 左侧树形列表 → 点击工位名称

3. 配置深度学习
   ├── ? 勾选 "是否加载模型"
   ├── 选择 "运行时类型"
   │   ├── GPU 运行时 (推荐新手)
   │   ├── OpenVINO 加速
   │   └── TensorRT 加速
 └── 选择 "模型文件夹路径"
       └── 浏览 → 选择包含 .hdl 和 .hdict 的文件夹

4. 系统弹出确认对话框
   ├── 显示当前配置信息
   └── 询问是否立即加载

5. 点击 [是]
   ├── 光标变为等待状态
   ├── 后台加载模型
   ├── 查看日志窗口（实时输出）
   └── 加载完成后弹出结果

6. 成功提示
   ? 模型加载成功！
   工位: 工位1
   模型: MyModel
   运行时: GPU 运行时
   
   [确定]

7. 保存方案
   └── 文件 → 保存方案
```

### 场景2: 切换运行时（性能优化）

```
问题：当前使用 GPU 运行时，推理速度慢

解决方案：
1. 打开工位配置
2. 选择工位
3. 运行时类型 → 改为 "TensorRT 加速"
4. 系统提示：是否立即加载模型？
5. 点击 [是]
   ├── ? 卸载旧模型（GPU）
   ├── ? 加载新模型（TensorRT）
   │   └── 注意：首次可能较慢（优化阶段）
   └── ? 加载成功

6. 测试性能
   └── 手动运行图片验证速度提升
```

### 场景3: 模型加载失败处理

```
1. 选择模型路径后点击 [是] 加载

2. 失败提示：
   ? 模型加载失败
   
   请查看日志窗口了解详细错误信息。
   
   常见问题：
   ? 模型文件不完整
   ? 运行时与硬件不匹配
   ? 缺少必要的驱动或库
   
   [确定]

3. 查看日志窗口
   └── 找到 ? 开头的错误信息
       └── 阅读详细错误原因和解决方案

4. 根据日志建议修复
   例如：
   ? 模型加载异常: 没有检测到可用设备
     → 错误原因: 未检测到支持的硬件设备
     → 当前运行时: TensorRT
     → 解决方案:
        ? 检查是否安装了 NVIDIA GPU 及 CUDA/TensorRT
        ? 尝试切换为 'GPU 运行时' 或 'OpenVINO 加速'

5. 按建议切换运行时
   └── 运行时类型 → 改为 "GPU 运行时"
       └── 再次尝试加载
```

## ?? 加载流程说明

### 加载前检查

```
系统自动检查：
├── ? 是否启用模型加载
├── ? 模型路径是否存在
├── ? 模型文件是否完整
│   ├── .hdl 文件（模型）
│   └── .hdict 文件（预处理参数）
└── ? DnnInferenceNet.dll 是否引用
```

### 加载过程

```
1. 卸载旧模型（如果存在）
   └── 调用 UnInit() 释放资源

2. 创建模型实例
   └── 反射加载 DnnSemanticSegmetation

3. 初始化模型
   ├── 参数1: 模型路径
   ├── 参数2: 设备类型 (GPU)
   └── 参数3: 运行时类型 (GC/OpenVINO/TensorRT)

4. 设备枚举
   ├── GC → 查找 GPU/CPU
   ├── OpenVINO → 查找 Intel 加速器
   └── TensorRT → 查找 NVIDIA GPU

5. 加载模型文件
   ├── 读取 .hdl (模型结构和权重)
   └── 读取 .hdict (预处理参数)

6. 设备绑定
   └── 将模型绑定到选定设备

7. 完成
   └── 模型实例保存到 station.DLModel
```

### 加载时间参考

| 运行时 | 首次加载 | 后续加载 | 说明 |
|--------|---------|---------|------|
| **GPU (GC)** | 2-5秒 | 2-5秒 | 稳定 |
| **OpenVINO** | 3-8秒 | 3-8秒 | 取决于模型 |
| **TensorRT** | 10-60秒 | 2-5秒 | 首次需优化 |

## ?? 最佳实践

### 1. 首次配置建议

```
推荐流程：
1. 先选择 "GPU 运行时"（兼容性最好）
2. 选择模型路径
3. 立即加载验证模型是否正常
4. 如果加载成功，再考虑优化运行时
```

### 2. 性能优化建议

```
测试流程：
1. 使用 GPU 运行时建立基准
2. 记录推理时间
3. 切换到 TensorRT（如有 NVIDIA GPU）
4. 等待首次优化完成
5. 对比性能提升
6. 如果满意，保存方案
```

### 3. 故障排查建议

```
加载失败时：
1. 不要立即关闭程序
2. 查看日志窗口
3. 根据建议切换运行时
4. 再次尝试加载
5. 仍然失败则尝试 GPU 运行时
```

## ?? 高级功能

### 手动重新加载

```
场景：模型文件更新了

方法1：切换运行时再切回
├── TensorRT → GPU → TensorRT
└── 触发重新加载

方法2：禁用再启用
├── 取消勾选 "是否加载模型"
├── 重新勾选
└── 触发重新加载

方法3：重新选择路径
└── 重新浏览选择同一路径
```

### 多工位批量配置

```
方法：
1. 配置第一个工位
   └── 立即加载验证成功

2. 配置其他工位
   ├── 使用相同配置
   └── 每个都立即加载验证

3. 全部成功后保存方案
```

## ?? 注意事项

### 1. 加载等待

```
? 加载过程中：
? 光标变为等待状态
? 界面可能短暂无响应
? 请耐心等待
? 不要关闭程序

特别提示：
TensorRT 首次加载可能需要 30-60 秒
这是正常现象，后续会很快
```

### 2. 内存占用

```
注意：
? 每个模型会占用一定显存/内存
? 如果配置多个工位，注意内存使用
? 建议只启用当前需要的工位
```

### 3. 保存方案

```
重要：
即时加载的模型在内存中
重启程序后需要重新加载

所以：
? 配置成功后立即保存方案
? 下次启动会自动加载
```
